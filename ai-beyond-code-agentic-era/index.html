12 / 28 Inside an AI Agent

What Makes an Agent Intelligent?

Goals

Knows what it's trying to achieve. Keeps focus even across many steps.

Memory

Remembers past actions, context, and results — short and long term.

Reasoning

Analyzes information, evaluates options, and decides what to do next.

Planning

Breaks complex goals into sequences of manageable sub-tasks.

Tools

Uses web search, code execution, email, calendars, APIs, databases.

Actions

Actually executes — sends emails, writes files, runs code, makes decisions.

These components work in a continuous loop until the goal is achieved

21 / 28 Responsibility Matters

Responsible AI — We Can't Skip This

AI Agents are powerful. Powerful tools need responsible hands.

Bias

AI trained on biased data produces biased results. Whose data? Whose voice?

Privacy

What data does the agent collect? Who owns it? Is it stored securely?

Security

Agents with access to email, files, and systems can be exploited if not secured.

Hallucinations

AI can confidently give wrong answers. Always verify critical information.

Human Oversight

For high-stakes decisions — health, law, finance — humans must stay in the loop.

Transparency

Users should know when they're talking to AI. No deceptive agents.

AI should assist — not replace — human judgment and human dignity.

25 / 28 The Code — Your First AI Agent in Python

20 Lines. A Real Agent.

# Step 1 — Install (run once in Colab)
!pip install groq -q

# Step 2 — Import & connect
from groq import Groq

client = Groq(api_key="YOUR_GROQ_API_KEY")

# Step 3 — Give the agent a role (system prompt)
history = [{
    "role": "system",
    "content": "You are a helpful study assistant for 
              Kabale University students. Help them 
              understand AI concepts clearly."
}]

# Step 4 — The agent loop (memory + reasoning)
while True:
    user_input = input("You: ")
    if user_input.lower() == "quit": break

    history.append({"role": "user", "content": user_input})

    response = client.chat.completions.create(
        model="llama-3.3-70b-versatile",
        messages=history
    )

    reply = response.choices[0].message.content
    history.append({"role": "assistant", "content": reply})
    print(f"Agent: {reply}\n")

What each part does

system prompt → defines the agent's role, personality, and constraints

history list → this IS the memory — every message is stored

while loop → the agent keeps running until you say "quit"

Groq API → free, fast, runs Llama 3.3 — no GPU needed
